## NekoHub 如何工作

NekoHub 由两个前端访问面和一套共享后端组成：

- **公开画廊**（`/gallery`）：匿名访问，只展示公开且已就绪的资产
- **管理后台**（`/login`、`/assets`、`/workflows`、`/providers`、`/users` 等）：通过用户名密码登录，使用 JWT 会话管理资产、用户、存储、AI 配置和 workflow
- **REST API / MCP**：供脚本、第三方应用和 Agent 工作流调用

<Steps>
  <Step title="部署 NekoHub">
    准备数据库连接、JWT 密钥、初始超级管理员账号，然后用 Docker Compose 启动。
  </Step>
  <Step title="登录管理后台">
    打开 `http://localhost:5173/login`，用首次部署时配置的账号密码登录。
  </Step>
  <Step title="上传第一张图片">
    进入 `/assets` 上传图片，可设置公开性、描述、替代文本以及是否运行 AI 技能。
  </Step>
  <Step title="共享或集成">
    公开资产可在 `/gallery` 展示；受保护接口可走 REST API；机器集成可走 API key 或 MCP。
  </Step>
</Steps>
